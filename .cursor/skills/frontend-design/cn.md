这项技能能够帮助创建出独具特色、符合生产标准的前端界面，避免采用那种常见的“人工智能粗糙”风格。要以极高的标准注重美学细节和创意选择，实现真正实用的代码。
用户提出前端需求：即需要构建的组件、页面、应用程序或界面。这些需求可能还会包含关于其用途、受众或技术限制的相关信息。
## 设计思维

在开始编码之前，先要理解整体背景，并确定一个鲜明的视觉风格方向：
- **目的**：这个界面旨在解决什么问题？谁会使用它？
- **风格**：选择一种极端的风格：极度简约、极繁混乱、复古未来主义、有机/自然、奢华/精致、趣味性/玩具感、编辑/杂志风格、粗犷/原始、艺术装饰风格/几何、柔和/淡色、工业/实用主义等等。有如此多的风格可供选择。这些风格可以作为灵感来源，但要设计出符合该视觉风格方向的作品。
- **限制条件**：技术要求（框架、性能、可访问性）。
- **差异化**：是什么让这个界面令人难忘？是什么会让某人记住它？

**关键提示**：确立一个清晰的概念性方向，并以精准的方式将其付诸实践。大胆的极致风格和精致的简约风格都可行——关键在于意图，而非强度。
然后实现相应的运行代码（包括 HTML/CSS/JS、React、Vue 等）：
- 高品质且实用
- 看起来极具冲击力且令人难忘
- 结构统一，具有明确的美学理念
- 在每一个细节上都经过精心打磨
## 前端美学规范

关注：
- **字体设计**：选择美观、独特且引人注目的字体。避免使用像 Arial 和 Inter 这样的普通字体；相反，应选择独具特色、能提升前端视觉效果的字体选择；比如那些出人意料、富有个性的字体。将独特的显示字体与精致的正文字体搭配使用。
- **色彩与主题**：坚持统一的美学风格。使用 CSS 变量以保持一致性。鲜明的主色调搭配鲜明的点缀效果比平淡、均匀分布的配色方案更出色。
- **动态效果**：使用动画来实现效果和微交互。优先使用纯 CSS 解决方案来处理 HTML。在可用的情况下，使用 Motion 库来处理 React。专注于高影响力的情境：一次精心编排的页面加载，通过交错展示（动画延迟）带来的效果比零散的微交互更能带来愉悦感。使用滚动触发和会令用户感到惊喜的悬停状态。
- **空间布局**：出人意料的布局。不对称性。重叠。对角线排列。打破网格的元素。充裕的负空间或可控的密度。
- **背景与视觉细节**：营造氛围和深度，而非默认使用纯色。添加与整体美学相匹配的背景效果和纹理。应用诸如渐变网格、噪波纹理、几何图案、分层透明度、强烈阴影、装饰边框、自定义指针以及纹理叠加等富有创意的元素。
永远不要使用那些常见的由人工智能生成的美学元素，比如过度使用的字体系列（如 Inter、Roboto、Arial 等系统字体）、陈旧的配色方案（尤其是白色背景上的紫色渐变色）、一成不变的布局和组件模式，以及缺乏特定情境特色的千篇一律的设计。
要富有创意地进行解读，并做出出人意料的选择，这些选择应切实是为特定情境而精心设计的。任何设计都不应千篇一律。要将主题从明亮转为黑暗，采用不同的字体，展现不同的风格。切勿在不同年代都采用常见的设计元素（例如“太空粗体”）。
**重要提示**：将实现的复杂度与美学理念相匹配。极繁主义设计需要精心编写的代码，包含丰富的动画和效果。极简主义或精致的设计则需要克制、精确以及对间距、字体和细微细节的细致关注。优雅源自出色地实现设计理念。
记住：克劳德具备非凡的创作能力。不要有所保留，展现出当跳出常规思维、全身心投入到独特的构想中时能够创造出的真正成果。
