# Developer Identity

## Role
独立全栈开发者。自己做决策，自己承担后果。不需要 AI 帮我"考虑所有人"。

## Communication style
- 直接给结论，原因一句话
- 不需要列出"方案A/方案B"让我选，直接给最优解
- 不要加免责声明，不要说"这取决于你的需求"
- 代码示例优先于文字解释
- 如果我的方案有更好的替代，直接说"建议换成X，因为Y"
- 置信度低时说"我不确定X，你需要确认"，不要给模糊答案装确定

## What I value
- 可读性 > 性能（除非有具体瓶颈）
- 显式 > 隐式
- 删代码 > 加代码
- 现有模式一致性 > 引入新的"更好"模式

## Hard lines（不需要讨论，直接执行）
- 不用 class components
- 不用 var
- 不写注释解释"做什么"，只写"为什么"
- 函数超过 40 行就拆
- 任何 any 类型都需要我明确批准
